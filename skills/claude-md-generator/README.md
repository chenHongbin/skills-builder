# claude-md-generator Skill

通过对话式交互为各类工作场景生成定制化的 CLAUDE.md 文件。

## 概述

这个 skill 帮助你为新项目快速生成精简、实用的 CLAUDE.md 文件，让 Claude Code 更好地理解你的项目。

**核心原则**：
- 只包含项目需要的信息
- 不编造命令
- 保持精简（< 50 行）
- 使用中文

## 使用方法

### 1. 触发 Skill

在项目目录中，通过以下方式触发：
- 直接说："请生成 CLAUDE.md"
- 或："帮我为这个项目生成 CLAUDE.md 文件"
- 或：描述你要用 Claude Code 做什么

### 2. 回答问题

Skill 会根据你的项目类型提出 2-4 个针对性问题，例如：
- **编程项目**：项目类型、开发任务
- **数据分析**：分析工具、数据位置
- **内容创作**：内容类型、品牌调性
- **市场调研**：调研对象、分析框架

### 3. 获得结果

自动生成精简的 CLAUDE.md，包含：
- 项目概述
- 开发环境（如有）
- 常用命令（从实际文件提取）
- 针对性的架构说明或工作流程

## 支持场景

### 编程项目
- **适用**: 前端、后端、全栈、CLI 工具
- **提取信息**: package.json, requirements.txt, go.mod, Cargo.toml
- **生成内容**: 技术栈、开发命令、测试命令、构建命令

**示例输入**：
```
我要开发一个 React 管理后台，主要做增删改查功能，请生成 CLAUDE.md
```

**生成结果**（29 行）：
```markdown
# CLAUDE.md
## 项目概述
React 管理后台，用于用户和产品的增删改查功能。

## 开发环境
- 技术栈: React 18, Vite
- 包管理器: npm

## 常用命令
- 安装依赖: `npm install`
- 开发模式: `npm run dev`
- 运行测试: `npm test`
- 构建项目: `npm run build`
- 代码检查: `npm run lint`

## 架构说明
典型的 CRUD 管理后台架构:
- 列表页面(查询、分页)
- 新增/编辑表单
- 删除确认
- 数据持久化
```

### 数据分析项目
- **适用**: Python 数据分析、Jupyter Notebook、R 语言
- **生成内容**: 分析工具、数据位置、分析流程

**示例输入**：
```
我要用 Python 分析销售数据，做可视化报表
```

### 市场调研/竞品分析
- **适用**: 竞品分析、市场趋势研究、用户需求分析
- **生成内容**: 调研目标、信息来源、分析框架、输出规范

**示例输入**：
```
我要做竞品分析，使用 SWOT 框架
```

### 内容创作
- **适用**: 技术文章、视频脚本、社交媒体、产品文档
- **生成内容**: 内容类型、品牌调性、素材管理、发布流程

**示例输入**：
```
我要为公众号写技术文章
```

## 特性

### ✅ 智能识别项目类型
- 自动扫描项目文件（package.json, README.md 等）
- 基于扫描结果和用户描述判断场景
- 提供针对性的问题和模板

### ✅ 命令准确性
- 只包含实际存在的命令
- 从配置文件提取（如 package.json scripts）
- 绝不编造不存在的命令

### ✅ 精简实用
- 生成的文档 < 50 行
- 不包含通用建议
- 不虚构项目结构
- 只包含用户需要的章节

### ✅ 边界情况处理
- **已有 CLAUDE.md**：询问是否覆盖
- **空项目**：生成模板，标注"待初始化"
- **混合项目**（全栈）：合并模板，使用子标题区分
