# 快商通录音分析

> 一通销售录音 → 本地转写 → 8 大版块分析 → 格式化的 Word 报告。把"听录音"从 1 小时变成 5 分钟。

## 概述

电销团队的管理者每周要听几十通录音——找亮点、抓问题、给辅导。大部分时间花在"听"和"记"上，而不是"思考"和"辅导"上。

这个 skill 把机械劳动自动化：
- **Whisper 本地转写**：录音数据不出电脑，私密安全
- **8 大版块标准化分析**：每通录音按同一框架诊断，方便横向对比
- **一键生成 Word 报告**：格式统一、可直接发给销售做辅导

**核心承诺：**
- 录音数据不外传——Whisper 在本地运行
- 每份报告都说人话——销售自己能看懂、知道怎么改
- 竞品严格区分——快商通≠商务通，绝不混淆
- 每条建议都可落地——"保持联系"是废话，"周二发案例"才是动作

## 何时使用

### 触发条件
- 用户提供了一段销售录音（.m4a/.mp3/.wav/.amr）并要求分析
- 用户说：分析录音、录音分析、帮我听下这通电话、生成录音报告、听下这个录音、分析下马龙锋这通电话
- 用户想批量分析多通录音做横向对比

### 何时不使用
- 用户只是想快速知道录音里说了什么（没有分析需求）——直接转写即可
- 用户提供的是视频文件（非音频）
- 录音不是销售场景（如内部会议、客户投诉等）

## 工作流程总览

```
用户提供录音文件
    │
    ▼
```

## 阶段 0：接收录音文件

### 0.1 定位录音文件

用户可能这样提供录音：
1. **直接给路径**：`/Users/xxx/Downloads/通话录音.m4a`
2. **只给销售姓名**：`分析马龙锋的最新录音`
3. **给微信文件夹路径**：`~/Library/Containers/com.tencent.xinWeChat/.../Message/...`

**查找策略：**
1. 如果用户给了完整路径，直接使用
2. 如果只给了姓名，询问用户录音位置：
   > "录音文件在哪个文件夹？通常在微信文件夹或下载文件夹里。"
3. 如果用户给出了微信 Messages 文件夹，用 `find` 搜索最近的录音文件：
   ```bash
   find "<微信Message路径>" -name "*.m4a" -o -name "*.mp3" -o -name "*.amr" -mtime -7
   ```

### 0.2 识别销售姓名和客户类型

从以下来源推断（按优先级）：
1. **用户明确告知**：`分析马龙锋的录音，客户是南宁贞韩医美`
2. **文件名推断**：文件名中可能包含姓名
3. **询问用户**：如果无法确定，问清楚再继续

### 0.3 确认目标目录

创建桌面上的「录音分析」文件夹结构：
```bash
mkdir -p ~/Desktop/录音分析/<销售姓名>/
```

**路径规则：**
- 桌面路径动态检测：`~/Desktop`（即 `/Users/$USER/Desktop/`）
- 每个销售一个子文件夹
- 向用户确认：`报告将保存在 桌面/录音分析/<姓名>/ 下`

## 阶段 1：音频转写

### 前置依赖检查（首次使用时）

在开始转写前，先检查必需工具是否已安装：

```bash
# 检查 ffmpeg
which ffmpeg > /dev/null 2>&1 && echo "✅ ffmpeg 已安装" || echo "❌ 请安装 ffmpeg：brew install ffmpeg"

# 检查 whisper（动态查找路径）
WHISPER=$(which whisper 2>/dev/null || find ~/Library/Python -name "whisper" -type f 2>/dev/null | head -1)
if [ -n "$WHISPER" ]; then
  echo "✅ whisper 已安装：$WHISPER"
else
  echo "❌ 请安装 whisper：pip3 install openai-whisper"
fi
```

**如果工具未安装，暂停并指导用户安装：**

| 工具 | 安装命令 | 说明 |
|------|---------|------|
| ffmpeg | `brew install ffmpeg` | macOS。Linux：`apt-get install ffmpeg` |
| openai-whisper | `pip3 install openai-whisper` | 本地转写，数据不外传 |

安装完成后继续。**不要跳过依赖检查**——没有这些工具无法完成转写。

### 1.1 格式转换

Whisper 对 wav 格式效果最好。先转换格式：

```bash
# 检查音频格式
ffprobe "<录音路径>" 2>&1 | grep -E "Stream|Duration"

# 如果不是 wav，转换为 wav（16kHz 单声道，Whisper 最佳输入）
ffmpeg -i "<录音路径>" -ar 16000 -ac 1 "/tmp/recording.wav" -y
```

### 1.2 Whisper 本地转写

使用 OpenAI Whisper 在本地转写（数据不出电脑）：

```bash
# 动态查找 whisper 路径
WHISPER=$(which whisper 2>/dev/null || find ~/Library/Python -name "whisper" -type f 2>/dev/null | head -1)

$WHISPER \
  "/tmp/recording.wav" \
  --model medium \
  --language zh \
  --output_dir /tmp \
  --output_format txt \
  --task transcribe
```

**关键参数说明：**
- `--model medium`：平衡速度与精度。small 中文识别不够准（销售术语多），large 太慢
- `--language zh`：强制中文识别
- `--task transcribe`：转写（非翻译）
- 转写结果保存在 `/tmp/recording.txt`

**转写时间预估：**
- 5 分钟录音：约 1-2 分钟处理
- 10 分钟录音：约 2-3 分钟处理
- 30 分钟录音：约 5-8 分钟处理

### 1.3 转写文本清理

```bash
python3 -c "
with open('/tmp/recording.txt', 'r') as f:
    text = f.read()

# 清理多余空行
import re
text = re.sub(r'\n{3,}', '\n\n', text)

# 标记说话人（如果有时间戳）
# Whisper 默认输出不含说话人标注，需要 AI 根据上下文推断

with open('/tmp/recording.txt', 'w') as f:
    f.write(text)

print(f'转写完成，共 {len(text)} 字符')
"
```

向用户汇报：`已转写完成，通话时长约 X 分钟，共 X,XXX 字。现在生成分析报告...`

## 阶段 2：8 大版块分析

> 阅读转写文本，按以下 8 个版块逐一分析。分析结果组织为 JSON 格式，供 generate_report.py 生成报告。

### 版块一：客户画像（基础信息与现状）

**分析任务：** 从对话中提取客户基本信息，判断是否值得重点跟进。

**AI 指令：**
```
从转写文本中提取并整理以下信息：
1. 客户姓名/机构/科室
2. 通话时长
3. 现有渠道/套电工具是什么
4. 每日对话量/留联率（如果有数据）
5. 团队规模
6. 判断：是否值得重点跟进？（标准：对话量100+/天 = 大客户）
```

**输出格式：**
```json
"customer_profile": {
  "basic_info": "客户名 | 机构 | 时长",
  "current_status": ["渠道：XX", "日对话量：XX条", "团队：X人"],
  "is_key_account": "判断结论和理由"
}
```

**判断标准：**
- 🟢 大客户：日对话 100+，有决策权，行业头部
- 🟡 中等客户：日对话 30-100，需要再确认决策人
- 🔴 小客户：日对话 < 30，或没有明确需求

---

### 版块二：通话核心：价值传递，诊断与算账

**分析任务：** 不仅记录聊了什么，更要看销售是否帮客户"算清了账"。

**AI 指令：**
```
分析本次通话中销售的价值传递：
1. 核心价值主张是什么？（如：捞回广告费、24小时不下班、提高留联率）
2. 销售是否帮客户算了 ROI？（有数字就算，没有则标❌）
3. 是否触及了医疗行业核心痛点？（夜间漏单、封号风险、客服接待慢、线索流失）
```

**输出格式：**
```json
"value_diagnosis": {
  "value_proposition": "销售传达的核心主张",
  "roi_calculation": "销售的算账内容（❌ 如果没有）",
  "pain_points": "触及的痛点列表"
}
```

**三个维度：**

| 维度 | 说明 |
|------|------|
| 价值传递 | 销售的核心主张是否清晰？一句话能说清楚吗？ |
| 数据算账 | 有没有具体数字？留联率提升X%→增收X万/月。有就算，无则标❌ |
| 医疗痛点 | 是否触及：夜间漏单、封号、接待慢、线索流失等。痛点挖得深不深？ |

---

### 版块三：客户意向与信号

**分析任务：** 用简单符号一眼判断成单概率。

**AI 指令：**
```
分析客户在通话中的意向信号：
1. 正面信号：问价格、问案例、要资料、答应测试、主动加微信...
2. 负面信号：嫌贵、要对比合同、说"再看看"、说"跟老板商量"...
3. 综合评级：🟢高（明确下一步） / 🟡中（有兴趣但犹豫） / 🔴低（敷衍或拒绝）
```

**输出格式：**
```json
"customer_intent": {
  "rating": "高|中|低",
  "positive_signals": ["信号1", "信号2"],
  "negative_signals": ["信号1"]
}
```

---

### 版块四：话术红黑榜

**分析任务：** 直接点出硬伤，给出示范话术。这是销售辅导最核心的部分。

**AI 指令：**
```
分析销售的话术：
1. 亮点（做得好 👍）：精彩的提问、精准的数据引用、成功的异议处理...
2. 硬伤（待改进 🔧）：每个问题给出具体改进话术

常见硬伤检查清单：
- 产品架构说不清（快商通≠商务通≠AI机器人，三者关系要一口清）
- 问句≠痛点（问"有没有了解"是功能性提问，追问"效果怎么样"才是挖痛）
- 价格无法当场报（单席3200/年，无限席43000/年，机器人按对话量）
- 口语化降低专业度（"牛逼"→"做得比较好"，"搞一下"→"处理一下"）
- 加微信前没埋价值钩子（要给客户一个接受微信的理由）
```

**输出格式：**
```json
"script_review": {
  "good_points": ["亮点1", "亮点2"],
  "improvements": [
    {"problem": "具体硬伤描述", "suggestion": "示范话术"}
  ]
}
```

**注意：** 指出问题时不要只批评——每条硬伤必须配一条"换成这样说"的示范话术。

---

### 版块五：竞争情报（竞品攻防）

**分析任务：** 识别竞品动态，找到快商通的打击点。

**关键规则：快商通 ≠ 商务通！二者是竞争关系，必须严格区分。**

**AI 指令：**
```
1. 客户现用什么工具？（商务通/巧三/人工/其他）
2. 客户对现有工具有什么不满？
3. 销售如何做竞品卡位？快商通 vs 竞品的针对性优势是什么？

如果销售混淆了快商通和商务通——必须标记为重大硬伤。
```

**输出格式：**
```json
"competitive_intel": {
  "current_tool": "客户现用什么",
  "dissatisfaction": "客户对现有工具的不满（原话）",
  "attack_points": ["快商通针对性优势1", "优势2"]
}
```

**快商通常见打击点参考：**
- 独家高德/抖音深转接口（竞品没有）
- 135 亿医疗语料训练（竞品没有）
- AI 套电率 60%+ vs 竞品 50%
- 24 小时自动响应 vs 竞品需人工值守

### 版块六：24h / 3d 关键动作

**分析任务：** 杜绝"保持联系"，必须是可落地的闭环动作。

**AI 指令：**
```
基于本次通话，制定下一步动作：
1. 24 小时内必须做什么？（具体到：发什么资料、拉什么群、确认什么信息）
2. 3 天内必须做什么？（具体到：哪天、几点、打电话还是发微信、说什么）
```

**输出格式：**
```json
"action_items": {
  "within_24h": ["具体动作1", "具体动作2"],
  "within_3d": ["具体节点和动作1", "具体节点和动作2"]
}
```

**反例与正例：**
- ❌ "保持联系，持续跟进"——说了等于没说
- ✅ "周二上午 10 点电话回访，问测试体验，准备好 XX 科室案例数据"

### 版块七：经理点评与总结

**分析任务：** 一通电话定性的总结，给管理者看的。

**AI 指令：**
```
给出三个维度的总结：
1. 成交可能性：0-100%
2. 一句话定性：这通电话的核心亮点或核心问题是什么？
3. 辅导重点：针对这个销售的改进优先级（如：竞品卡位 > 价值算账 > 加微信话术）
```

**输出格式：**
```json
"manager_summary": {
  "closing_probability": 75,
  "one_liner": "一句话定性",
  "coaching_priority": "辅导优先级排序"
}
```

## 阶段 3：生成 Word 报告

### 3.1 组织分析结果为 JSON

将阶段 2 的 8 个版块分析结果合并为一个完整的 JSON：

```bash
# 将分析结果写入临时 JSON 文件
cat > /tmp/analysis_result.json << 'ANALYSIS_JSON'
{
  "sales_name": "马龙锋",
  "date": "26.5.13",
  "customer_type": "医美",
  "output_path": "/Users/$USER/Desktop/录音分析/马龙锋/",
  "sections": { ... }
}
ANALYSIS_JSON
```

### 3.2 调用报告生成脚本

```bash
python3 ~/.claude/skills/kuaishangton-recording-analysis/scripts/generate_report.py /tmp/analysis_result.json
```

### 3.3 文件命名规则

- 单通：`日期_姓名_客户类型_录音分析.docx`
  - 示例：`26.5.13_马龙锋_医美_录音分析.docx`
- 多通汇总：`日期_姓名_多通汇总_录音分析.docx`
- 日期格式：`年.月.日`（如 26.5.13）

### 3.4 文档格式规范

- 页眉：快商通 · 电销团队录音分析报告
- 正文字体：Arial 11pt
- 标题层级：H1（版块标题）、H3（子标题）
- 评级用 emoji：🟢🟡🔴✅❌👍🔧
- 页脚：快商通 · AI 驱动销售增长

### 3.5 向用户汇报

```
✅ 录音分析完成！

📄 报告已保存：桌面/录音分析/马龙锋/26.5.13_马龙锋_医美_录音分析.docx

📊 核心摘要：
- 客户意向：🟢 高
- 成交概率：75%
- 最大亮点：价值算账清晰，客户沉默了
- 最大硬伤：竞品卡位话术没接住
- 下一步：24h 内发同城案例 + 约测试

辅导优先级：竞品卡位 > 加微信话术 > 价值算账
```

## 多通录音批处理

当用户一次提供多通录音时：

1. **逐通转写**：每通录音独立转写，存为 `/tmp/recording_1.txt`、`/tmp/recording_2.txt`...
2. **逐通分析**：每通按 8 大版块分析
3. **生成综合对比表**（版块八）
4. **生成汇总报告**：一份 Word 文档包含所有录音的分析 + 对比表
5. 文件命名：`日期_姓名_多通汇总_录音分析.docx`

**批量分析时向用户确认：**
> "检测到 X 通录音。每通预计需要 3-5 分钟处理，总共约 XX 分钟。是否继续？"

## 边界情况

### 1. 录音转写失败
- Whisper 崩溃或超时 → 告知用户，建议手动转写或检查音频文件
- 音频质量太差（背景噪音大、声音模糊）→ 告知用户转写可能不准确，仍继续分析但标注"转写质量受限"
- 非中文内容过多 → 去掉 `--language zh` 参数让 Whisper 自动检测

### 2. 无法识别销售人员
- 文件名不含姓名、用户也没说 → 用 AskUserQuestion 询问
- 多个销售同用一个文件夹 → 请用户确认是哪位

### 3. 录音内容不完整
- 只录了一半 → 分析时可用的部分，标注"通话不完整"
- 只有一方声音 → 可能只录了客户或销售一侧

### 4. 非销售场景录音
- 内部会议、客户投诉等 → 告知用户这通录音不适合用 8 大版块分析，询问是否改为普通转写

### 5. 桌面/录音分析目录不存在
- 自动创建：
  ```bash
  mkdir -p ~/Desktop/录音分析/<销售姓名>/
  ```

## 常见反模式（不要做的事）

| 反模式 | 为什么失败 | 修复方法 |
|--------|-----------|---------|
| **只记录不诊断** | 客户画像只写了基本信息，没说"值不值得跟" | 每个版块必须有判断结论 |
| **批评不示范** | 只说"这里说得不好"，没给改进话术 | 每条硬伤配一条示范话术 |
| **竞品混淆** | 把快商通和商务通说成一家 | 严格区分，混淆是重大硬伤 |
| **动作说废话** | "保持联系""继续跟进" | 必须是 X 时间做 Y 动作、发给谁、说什么 |
| **数据模糊** | "效果不错""很多客户" | 必须具体数字：提升 X%、增收 X 万 |
| **漏版块** | 8 大版块不全 | 每通录音必须 8 个版块全覆盖 |
| **转写外传** | 把录音上传到第三方 API 转写 | 只用本地 Whisper |

## 禁止事项

**绝对不要做的事：**
- ❌ 把录音上传到任何第三方 API 进行转写（隐私红线）
- ❌ 混淆快商通和商务通（竞品关系，必须严格区分）
- ❌ 漏掉任何一个版块（8 大版块必须全覆盖）
- ❌ 给出模糊的跟进动作（"保持联系"是禁用词）
- ❌ 只指出问题不给示范话术
- ❌ 在报告中写"牛逼""搞一下"等口语（除非是引用客户原话）
- ❌ 跳过确认环节直接保存报告

**每次对话开始时的验证：**
- [ ] 录音文件路径已确认
- [ ] 销售姓名和客户类型已确认
- [ ] Whisper 和 ffmpeg 工具可用
- [ ] 目标目录已创建
- [ ] 转写完成后才开始分析
- [ ] 8 大版块全覆盖
- [ ] 报告已成功保存并告知用户路径
