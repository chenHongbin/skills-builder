# 反模式与禁止事项

## 常见反模式

| 反模式 | 为什么失败 | 修复方法 |
|--------|-----------|---------|
| **只记录不诊断** | 摘要只写了基本信息，没说"值不值得跟""问题在哪" | 每个维度必须有判断结论 |
| **评分没理由** | 只给数字不说为什么 | 每个分数必须配一句话理由 |
| **批评不示范** | 只说"这里说得不好"，没给改进话术 | 每条硬伤配一条示范话术 |
| **动作说废话** | "保持联系""继续努力" | 必须是 X 时间做 Y 动作、说什么话 |
| **数据模糊** | "效果不错""很多客户" | 必须具体数字：提升 X%、增收 X 万、X% 的客户 |
| **漏维度** | 6 大维度不全 | 每通录音必须全覆盖 |
| **转写外传** | 把录音上传到第三方 API 转写 | 只用本地 Whisper |
| **分析变成批评大会** | 只指出问题不肯定亮点 | 必须有红榜（亮点），销售才知道怎么复制 |
| **团队诊断只看平均** | 只说"团队平均分 6.5"，不说谁高谁低 | 必须给出标杆、短板、关注名单 |
| **批量分析不汇总** | 分析了 20 通但只给 20 份单通报告 | 必须生成团队汇总诊断和横向对比 |

## 绝对禁止事项

- ❌ **把录音上传到任何第三方 API 进行转写**（隐私红线）
- ❌ **漏掉任何一个维度**（6 大维度必须全覆盖）
- ❌ **给出模糊的跟进动作**（"保持联系"是禁用词）
- ❌ **只指出问题不给示范话术**
- ❌ **评分不给理由**
- ❌ **在报告中写"牛逼""搞一下"等口语**（除非是引用客户原话）
- ❌ **跳过确认环节直接保存报告**
- ❌ **绑定任何具体产品**（本 skill 是通用的，不能出现"快商通""商务通"等特定产品名）
- ❌ **批量分析时不生成团队汇总**（只给单通报告等于没做批量分析）
- ❌ **忽视趋势变化**（本周 vs 上周的对比是管理洞察的核心）

## 每次对话开始时的验证清单

- [ ] 录音文件路径已确认（单通）或文件列表已汇总（批量）
- [ ] 销售姓名和客户类型已确认（如无法确认则标注"未知"）
- [ ] Whisper 和 ffmpeg 工具可用
- [ ] 目标目录已创建
- [ ] 转写完成后才开始分析
- [ ] 6 大维度全覆盖
- [ ] **批量分析时：团队诊断和横向对比已生成**
- [ ] 报告已成功保存并告知用户路径
