# 边界情况处理

## 1. 录音转写失败

| 情况 | 处理方式 |
|------|---------|
| Whisper 崩溃或超时 | 告知用户，建议检查音频文件或手动转写 |
| 音频质量太差（噪音大、声音模糊） | 告知用户转写可能不准确，仍继续分析但标注"转写质量受限" |
| 非中文内容过多 | 去掉 `--language zh` 参数让 Whisper 自动检测 |
| 录音文件损坏 | 尝试用 ffprobe 检测，如确认损坏则告知用户无法处理 |

## 2. 无法识别销售人员

| 情况 | 处理方式 |
|------|---------|
| 文件名不含姓名、用户也没说 | 用 AskUserQuestion 询问 |
| 多个销售同用一个文件夹 | 请用户确认是哪位，或按文件名前缀推断 |
| 用户只说"分析最近的录音" | 按文件修改时间排序，取最近 1-3 通，向用户确认 |

## 3. 录音内容不完整

| 情况 | 处理方式 |
|------|---------|
| 只录了一半 | 分析可用的部分，标注"通话不完整" |
| 只有一方声音 | 可能只录了客户或销售一侧，标注并继续分析 |
| 开头/结尾缺失 | 重点分析中间可用部分，缺失部分标注"录音不完整" |

## 4. 非销售场景录音

| 情况 | 处理方式 |
|------|---------|
| 内部会议 | 告知用户不适合用本框架分析，询问是否改为普通转写 |
| 客户投诉 | 告知用户这是投诉场景，建议用投诉处理框架而非销售分析框架 |
| 技术支持/售后 | 告知用户这是服务场景，分析维度不同，询问是否继续 |
| 完全无关内容 | 直接告知用户录音内容与销售无关，无法分析 |

## 5. 批量分析时的特殊情况

| 情况 | 处理方式 |
|------|---------|
| 录音数量超过 50 通 | 建议分批处理，或先按日期/销售筛选 |
| 录音分布在多个文件夹 | 汇总所有路径，统一处理 |
| 部分录音转写失败 | 跳过失败的，继续分析成功的，最后报告哪些未成功 |
| 团队录音中混入了非销售录音 | 识别出后标注，不计入团队诊断统计 |

## 6. 目录和文件问题

| 情况 | 处理方式 |
|------|---------|
| 桌面/录音洞察目录不存在 | 自动创建：`mkdir -p ~/Desktop/录音洞察/` |
| 销售姓名子文件夹不存在 | 自动创建：`mkdir -p ~/Desktop/录音洞察/<姓名>/` |
| 文件名冲突 | 追加序号：`日期_姓名_客户类型_录音洞察_1.docx` |
| 磁盘空间不足 | 转写前检查 /tmp 和桌面空间，不足时告知用户 |
