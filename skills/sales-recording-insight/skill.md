# 电销录音管理教练

> 把销售对话从"听感经验"变成"可分析、可复盘、可训练的数据资产"。
> 管理者视角，批量分析，团队诊断，数据驱动。

## 核心设计

**默认假设：用户是销售管理者，通常要分析多通录音。**
单通分析是子流程，批量团队分析是核心能力。

**分析后必须回答的管理问题：**
1. 这通/这批电话里，客户说了什么？销售做了什么？
2. 谁的通话质量好，谁的问题最多？
3. 团队共性短板是什么？本周训练主题是什么？
4. 哪些客户值得重点跟进？哪些被浪费了？
5. 明天一对一辅导谁、说什么？

**核心承诺：**
- 录音数据不外传——只用本地 Whisper
- 每份报告聚焦"管理动作"——不仅分析电话，更告诉你怎么管人
- 批量分析必出团队诊断——只给单通报告等于没做批量分析
- 通用不绑定产品——任何行业的电话销售都适用

---

## 工作流程

### 批量分析流程（默认）

```
用户提供录音（多通/文件夹/日期范围）
    │
    ▼
阶段 0：汇总录音清单
    - 扫描文件夹，收集所有录音文件
    - 按销售姓名分组
    - 确认分析范围（多少通、哪些人、什么时间段）
    │
    ▼
阶段 1：逐通转写
    - 每通独立转写为 /tmp/recording_N.txt
    - 汇总转写统计（总时长、总字数）
    │
    ▼
阶段 2：逐通分析（6大维度）
    - 每通按同一框架分析，生成独立 JSON
    │
    ▼
阶段 3：团队汇总诊断（批量核心）
    - 横向对比：谁强谁弱、进步退步
    - 共性诊断：团队短板、训练主题
    - 关注名单：谁需要一对一、谁需要提醒
    - 趋势变化：本周 vs 上周/上月
    │
    ▼
阶段 4：生成报告
    - 团队汇总报告（Word）
    - 单通详情作为附录或独立文件
    │
    ▼
汇报：返回核心摘要 + 管理动作清单
```

### 单通分析流程（子流程）

当用户明确只分析一通录音时，跳过"团队汇总"阶段，直接生成单通报告。
但输出格式仍需包含"管理洞察"维度（辅导优先级、一对一话术等）。

---

## 阶段 0：接收录音

### 0.1 定位录音文件

**批量场景：**
```bash
# 扫描指定目录下的所有录音
find "<目录路径>" -type f \( -name "*.m4a" -o -name "*.mp3" -o -name "*.wav" -o -name "*.amr" \) -mtime -7

# 按修改时间排序，统计数量
ls -lt "<目录路径>"/*.{m4a,mp3,wav,amr} 2>/dev/null | wc -l
```

**单通场景：**
用户直接给路径，或使用 find 搜索最近的录音。

**确认分析范围：**
批量时向用户确认：
> "找到 X 通录音，来自 Y 位销售，时长总计约 Z 分钟。预计分析时间约 W 分钟。是否继续？"

### 0.2 识别销售和客户信息

- 从文件名推断销售姓名（如 `小李_某客户_0520.m4a`）
- 从文件名推断客户类型
- 无法推断时标注"未知"

### 0.3 创建目标目录

```bash
mkdir -p ~/Desktop/录音洞察/团队/
mkdir -p ~/Desktop/录音洞察/<销售姓名>/
```

## 阶段 1：音频转写

### 前置依赖检查

```bash
which ffmpeg > /dev/null 2>&1 && echo "✅ ffmpeg" || echo "❌ brew install ffmpeg"
WHISPER=$(which whisper 2>/dev/null || find ~/Library/Python -name "whisper" -type f 2>/dev/null | head -1)
[ -n "$WHISPER" ] && echo "✅ whisper: $WHISPER" || echo "❌ pip3 install openai-whisper"
```

未安装则暂停并指导安装。

### 1.1 格式转换与转写

```bash
# 转单通
ffmpeg -i "<录音路径>" -ar 16000 -ac 1 "/tmp/recording.wav" -y
$WHISPER "/tmp/recording.wav" --model medium --language zh --output_dir /tmp --output_format txt --task transcribe

# 清理文本
python3 -c "
import re
with open('/tmp/recording.txt', 'r') as f: text = f.read()
text = re.sub(r'\n{3,}', '\n\n', text)
with open('/tmp/recording.txt', 'w') as f: f.write(text)
print(f'转写完成，{len(text)} 字符')
"
```

**批量时：** 循环处理每通录音，转写结果存为 `/tmp/recording_1.txt`, `/tmp/recording_2.txt`...

## 阶段 2：逐通分析（6大维度）

对每通录音，阅读转写文本，按以下 6 个维度分析。结果组织为 JSON。

### 维度一：通话摘要

提取核心信息：
1. 客户是谁（姓名/公司/职位）
2. 客户需求是什么
3. 客户异议是什么
4. 销售讲了什么核心内容
5. 结果是什么（具体到动作：约周三演示 / 发资料 / 再考虑）
6. 通话时长

输出：
```json
"call_summary": {
  "customer": "...", "need": "...", "objection": "...",
  "sales_pitched": "...", "outcome": "...", "duration": "X分X秒"
}
```

### 维度二：关键标签

给通话打三类标签：

**核心标签**（必选一个）：
- `高意向客户`——主动问价/方案/合同/付款
- `成交信号`——表达购买意愿或确认下一步
- `跟进机会`——未拒绝，需后续推动
- `无效沟通`——无实质内容或客户敷衍

**问题标签**（多选）：
`价格异议` `功能异议` `竞品对比` `决策人未触及` `销售失误`

**能力标签**（多选）：
`需求挖掘充分` `异议处理到位` `价值传递清晰` `推进动作明确`

输出：
```json
"tags": { "core": ["..."], "problems": ["..."], "strengths": ["..."] }
```

### 维度三：话术评分

从 5 个维度评分，满分 10 分，每个分数配一句话理由。

| 维度 | 考察点 |
|------|--------|
| 开场有效性 | 是否快速建立信任、说明来意、激发兴趣 |
| 需求挖掘充分度 | 是否问出真实需求、痛点、预算、决策流程 |
| 异议处理能力 | 面对价格/功能/竞品异议时处理是否得当 |
| 价值表达清晰度 | 产品价值是否说清楚，客户能否听懂 |
| 推进下一步明确度 | 结尾是否有明确的下一步动作（时间/地点/方式） |

**评分标准：** 9-10优秀/7-8良好/5-6及格/3-4较差/1-2很差

**综合得分：** 短板维度权重更高。若"推进下一步"仅 5 分，即使其他高，综合不超过 7 分。

输出：
```json
"scores": {
  "opening": {"score": 8, "reason": "..."},
  "need_discovery": {"score": 6, "reason": "..."},
  "overall_score": {"score": 6.8, "level": "良好但有明显短板"}
}
```

### 维度四：话术红黑榜

**红榜（亮点）：** 精彩的提问、成功的异议处理、精准的价值表达、有效的推进话术。每条标注"在哪个时刻"+"做了什么"。

**黑榜（硬伤）：** 每条包含：
- 问题描述（具体，不是"说得不好"）
- 示范话术（完整可替换的话术，不是"应该更委婉"）

常见硬伤：开场废话多、问句太浅、自说自话、异议处理对抗、价值模糊、结尾没动作、过度承诺、打断客户。

输出：
```json
"script_review": {
  "good_points": [{"moment": "...", "what": "..."}],
  "improvements": [{"moment": "...", "problem": "...", "suggestion": "..."}]
}
```

### 维度五：改进建议

输出三层可执行建议：

1. **立即改进**（下次就能用）：具体动作 + 示范话术
2. **短期提升**（本周练）：能力点 + 练习方式 + 验收标准
3. **长期成长**（持续优化）：方向 + 方法

输出：
```json
"actionable_advice": {
  "immediate": {"action": "...", "script": "..."},
  "short_term": {"focus": "...", "practice": "...", "success_criteria": "..."},
  "long_term": {"direction": "...", "method": "..."}
}
```

### 维度六：管理洞察

从管理者视角输出：
1. 核心问题（一句话）
2. 辅导优先级排序
3. 客户是否值得跟进 + 理由
4. 一对一辅导话术（给主管参考，明天说什么）
5. 团队经验（亮点可复制 / 问题需警惕）

输出：
```json
"management_insight": {
  "core_problem": "...",
  "coaching_priority": ["...", "..."],
  "customer_worth": {"judgment": "...", "reason": "..."},
  "one_on_one_script": "...",
  "team_lesson": {"positive": "...", "warning": "..."}
}
```

## 阶段 3：团队汇总诊断（批量分析必做）

当分析多通录音时，在所有逐通分析完成后，必须生成团队汇总。

### 3.1 横向对比

对比维度：
- 综合评分排名（谁最高/最低）
- 各维度评分对比（谁在开场强、谁在异议处理弱）
- 标签分布统计（多少通是高意向、多少通有价格异议）
- 进步/退步趋势（如有历史数据）

### 3.2 共性诊断

输出：
```json
"team_diagnosis": {
  "strengths": ["团队整体..."],
  "weaknesses": ["80% 的销售...", "一半的人..."],
  "training_theme": "本周团队训练主题：...",
  "attention_list": ["小张连续 3 通低于 4 分", "小李话术亮点最多，可内部分享"],
  "highlights": ["小王开场值得学习", "小刘异议处理进步大"]
}
```

### 3.3 管理动作清单

基于团队诊断，直接输出管理者明天要做什么：

```
📋 明日管理动作：
- 早上：把 X 个高意向客户分配给对应销售，确认跟进状态
- 中午：15 分钟短会，放 XX 的录音（标杆学习 / 典型问题）
- 晚上：一对一辅导 XX，练 XX 话术
```

## 阶段 4：生成报告

### 4.1 组织 JSON

将分析结果合并为 JSON，调用报告生成脚本：
```bash
python3 ~/.claude/skills/sales-recording-insight/scripts/generate_report.py /tmp/analysis_result.json
```

### 4.2 文件命名

- 单通：`日期_姓名_客户类型_录音洞察.docx`
- 批量汇总：`日期_团队_周汇总_录音洞察.docx`
- 日期格式：`年.月.日`（如 26.5.20）
