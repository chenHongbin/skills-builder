# Skill 改进点识别

## 基于测试反馈的改进需求

### 1. 强化用户交互要求

**问题**: Subagent 没有使用 AskUserQuestion
**原因**:
- 用户描述已经很详细
- package.json 提供了充分信息
- Subagent 认为不需要额外提问

**解决方案**:
在 skill 中添加更强的指令：
- 即使信息充分，也要提问确认
- 提问可以了解用户偏好和优先级
- 明确标注"必须使用 AskUserQuestion"

### 2. 添加禁止事项清单

**问题**: 需要更明确的"不要做"清单
**原因**: 防止 subagent 添加不必要的内容

**解决方案**:
添加"禁止事项（严格遵守）"部分：
- 绝对不要编造命令
- 绝对不要添加通用建议
- 绝对不要虚构项目结构
- 提供验证清单

### 3. 优化 CSO（Claude Search Optimization）

**问题**: 确保 skill 能被正确触发
**原因**: description 需要包含触发关键词

**解决方案**:
- 确认 description 简洁明确
- 在文档中添加关键词：项目初始化、新项目设置、CLAUDE.md 创建

### 4. 测试其他场景

**当前**: 只测试了 React 项目（有 package.json）
**需要**: 测试其他场景
- 数据分析项目（空目录）
- 内容创作项目（空目录）
- 混合项目（全栈）

5. 保持精简（< 50 行）
